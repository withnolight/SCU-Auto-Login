# SCU Auto Login (四川大学教务处自动登录)

这是一个用于四川大学教务系统的自动登录工具，包含一个基于 `ddddocr` 的验证码识别后端和一个 Tampermonkey 用户脚本。

## 快速安装

[Get on Greasy Fork](https://greasyfork.org/zh-CN/scripts/565585-scu-auto-login)


## 功能特性

- 自动填充学号和密码
- 自动识别并填写验证码
- 支持配置多个 OCR 服务器地址
- 识别时并发请求全部 OCR API，首个成功响应自动填入并终止其余请求

## 环境要求

- Python 3.11 (无法在3.13及以上版本运行)
- 浏览器 (Chrome/Edge/Firefox)
- Tampermonkey 插件

## 安装与使用

### 1. 启动 OCR 后端服务

后端服务用于识别登录页面的验证码。

0.  目前此项目已经升级为使用 `pixi` 作为版本控制工具，推荐使用 `pixi` 来管理项目依赖和运行环境。如果你还没有安装 `pixi`，请先安装它。[安装教程](https://hellowac.github.io/pixi-zh-cn/#_2)
1.  克隆或下载本项目。
2.  安装依赖：
    ```bash
    pixi install
    ```
3.  启动服务器：
    ```bash
    pixi run server
    ```
    服务默认运行在 `http://127.0.0.1:5000`。

### 2. 安装用户脚本

1.  确保浏览器已安装 Tampermonkey 插件。
2.  新建一个脚本，将 `SCU-Auto-Login.user.js` 的内容复制进去并保存。
3.  或者直接在 Tampermonkey 管理面板中导入文件。

### 3. 配置与登录

1.  打开四川大学教务系统登录页面 ( http://zhjw.scu.edu.cn/login )。
2.  脚本首次运行会弹出提示框，要求输入配置信息：
    - **学号**: 您的 SCU 学号
    - **密码**: 您的教务系统密码
    - **OCR 服务器地址（可多个）**: 支持输入多个地址，使用英文逗号或换行分隔。示例：
        ```
        http://127.0.0.1:5000/api/ocr
        https://example.com/api/ocr
        ```
      脚本会并发请求所有地址，谁先成功返回验证码就使用谁，并自动取消其余请求。
    > 目前可用服务器地址：
    > | 提供者 | 地址 | 备注 |
    > |:------|:------|:------|
    > | withnolight | https://succina.cc.cd/api/ocr | 服务器位于美国达拉斯，延迟较大，无法做到无感登录|

    > 如果你愿意提供有多余的服务器资源，欢迎在issue里一起分享！

3.  配置完成后，刷新页面，脚本将自动完成登录过程。

### 4. 修改配置

如果需要修改已保存的学号、密码或服务器地址，请访问以下链接触发重新配置：
- http://zhjw.scu.edu.cn/login?config=1

或者在 URL 后手动添加 `?config=1` 参数并刷新页面。

## 常见问题

- **无法连接服务器**: 请确保 `server.py` 正在后台运行，且端口 5000 未被占用。
- **配置了多个服务器但仍未登录**: 只有返回 `success` 且包含验证码结果的响应才会被采用；如果全部接口失败或返回无效结果，请检查接口可用性与返回格式。
- **验证码识别率低**: `ddddocr` 是通用 OCR 模型，可能存在少量识别错误，刷新验证码重试即可。
- **跨域问题**: 后端已配置 `flask-cors`，通常不会遇到跨域错误。
